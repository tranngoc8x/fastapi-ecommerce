# Python Package Structure Standards [L1]

## __init__.py Files
- Clear purpose
- Module docstrings
- Version info
- Component registration
- No side effects
- Package validation
- Nested structure
- Integrity checks

## Import Management
- Public interface (__all__)
- Explicit exposure
- Namespace flattening
- Convenient re-exports
- Lazy imports
- Clear subpackages
- No cyclic imports
- Relative imports
- Minimal dependencies
- Clean hierarchy
- Internal protection

## Type Hints
- Stub files (.pyi)
- Clear annotations
- Import types
- Generic support
- Protocol definitions
